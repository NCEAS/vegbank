Vegetation Plots Database - working draft 7/30/2000
 

plot_descriptive_attributes

plot_id - System assigned unique identifier for each individual plot.  At this time a vegetation plot is defined by both discrete location (a single point) and date; meaning that two plots may exist for a single location if the supporting data were collected at differing times.  Also, sub-plots having smaller lateral coverage may exist within a larger regional plot.

altmin - minimum altitude within a plot.  Because a vegetation plot has been defined as a single point in time and space, we may want to store just the altitude at the origin of the plot.  On the other hand, I recognize that a plot does have area and the vegetation within that area is seen as a proxy for a discrete point - so at this point i'm in favor of storing the min and max altitude just as I hope to store the coordinates of the sampling area

altmax - altitude maximum (see above for comments)

plotOrigin - The discrete geographic point defining the plot.  This attribute will be represented by a latitude and longitude

dsgpolyo -  Data Set G-Polygon coordinates defining the outline of an area covered by a data set from the csdgm-fgdc-std-001-1998.pdf.  Even though a plot is defined as a single point, we recognize that the point is reflective of a sampling area and storing that area is important.  This information does not have to be collected directly from the user; instead the points will be calculated by the interface or database.  Many plots, for example, are circles whose center is the plotOrigin. Prompting the user for the shape of the plot, most plots are circular or rectilinear, the plotOrigin and, in this case, radius we can calculate this polygon.  In the case of closing polygons (circles, rectangles, etc.) it is suggested that the first and last pair of coordinates defining a point in a polygon have the same location.  Plots defined by non-closing polygons (lines) will obviously not have this condition.

town - town in which the plot was collected

county - county which the town is in

state - state in which the county exists

country - country where the plot was collected (these location identifiers are used, for among other reasons, to detect major errors in the lat./long. records

altdatum    -    Altitude Datum consistent with csdgm-fgdc-std-001-1998.pdf

altres    -    Altitude Resolution consistent with csdgm-fgdc-std-001-1998.pdf

altunits    -     Altitude Units consistent with csdgm-fgdc-std-001-1998.pdf

altenc    -    Altitude Encoding Scheme consistent with csdgm-fgdc-std-001-1998.pdf

slope_aspect  -  Azimuth of the Slope consistent with the  FGDC-STD-006 Soil Geographic Data Standard (word document)  At this point only one slope description per plot is going to be stored.

slope_gradient    -    Gradient of the Slope consistent with the  FGDC-STD-006 Soil Geographic Data Standard (word document)

location_confidentiality - Security Attribute - adopted from the iso-geo-md-1999Dec-draft.pdf 'MD_Classification'

hydrologic_regime - Soil element (see FGDC-STD-006 Soil Geographic Data Standard) follows Cowardin et al. 1979 - see vegclass.pdf, includes tidal and non-tidal zones

soil_moisture    -    This follows Hills' (10 step) moisture regime scale

soil_drainage    -    This follows Hills' (10 step) moisture regime scale

water_table_depth - (not added yet) depth to water table in meters

geological_materials    -    not yet defined - but will probably be type of bedrock and surficial deposits

other_conditions    -    not yet defined (waiting to hear from ecologists)
