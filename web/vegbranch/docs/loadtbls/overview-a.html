VegBranch Loading Tables Overview -- Party Information (A)

Navigation between loading tables:

Summary: Information about the different people (parties) involved in your plots, projects, taxa, and classifications.
Description: Party data has a UserPartyCode which can then be used to identify any party in other loading tables. This code can be any combination of letters or numbers. Firstname plus lastname would make a good code if you have no codes already for parties.
This table allows you to enter data about each party, such as names, organization, address, and phone number.
Required in VegBranch: Not required. VegBranch can generate this table from other tables (Plot Data and Plot Community Data) with the Party Form or Metadata Form.
Represents VegBank Tables: party, address (1), telephone (1)
VegBranch can generate for you: VegBranch can generate this table from the contrib fields in Plot Data (or Plot Normalized) and the InterpParty fields in Plot Communities.
Questions: Who participated in observing or classifying plots?

Fields (Columns) in this table:

More information about fields available here.

» Required fields; you should always fill in values for these:
UserPartyCode A unique code that will refer to a party consistently throughout the loading process. Other loading tables will use this code to reference the party of this record. The code can be any combination of letters and numbers. If you don't have a code for parties, you could simply use the first name and the last name for a code.
» Commonly used fields:
SurName Name shared in common to identify the members of a family, as distinguished from each member's given name.
GivenName One's first name.
OrganizationName Name of an organization.
orgPosition Position of party within organization.
Email email address of party member
» Sometimes used fields:
middleName One's middle name or initial, if any.
contactInstructions Instructions for contacting a party.
TelephoneNumber Telephone number.
TelephoneType This is the type of telephone: home, work, fax, cell, secretary, other.
deliveryPoint Address line for the location: for example, Street name, box number, suite.
city City of the location.
administrativeArea State, province of the location.
postalCode Zip code or other postal code.
country Country of the physical address.
currentFlag This flag signifies whether the address is the current address of the party. "True" is used for addresses that are current.
addressStartDate The first database date on which the address/organization information applied (have a discrete date on which address is known to applied -- not necessarily the first date on which the address applied).
partyType The type of party this is. Person is the most common type.
» Internal VegBranch fields, please do not edit directly:
Party_ID Database generated identifier assigned to each unique party.
VegBankParty_ID The Party_ID of this Party that has been migrated into VegBank module.
VegBankAccCode Code that uniquely references the party corresponding to this record, allowing reference to this record for this version of this database. Better than a primary key, which is automatically generated by a database and not globally unique. VegBank Accession codes are only generated by VegBank, and therefore data integrity can be guaranteed, unlike with a primary key (number).
