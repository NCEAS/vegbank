VegBranch Loading Tables Overview -- Plot Communities (I)

Navigation between loading tables:
Party Information | Species List | Plot Data | Plot Normalized | Presence Data | Strata Cover Data | Stem Data | Stem Classes | Soil Data | Disturbance Data | Community Concept | Plot Communities | User Defined Metadata

Jump to a different part of this page: Field Descriptions | Example Data

Summary: Assigns plots to particular communities (up to 4 per record), based on a methodology that you specify.
Description: Plots may be assigned to communities with this table. Up to four communities may be linked to a plot with commName1-commName4. These are matched to VegBank concepts in either the Community Loading Table or by linking directly to a community already in the vegbank_module (with a standard name, ie CEGL code). Plots are linked with authorPlotCode.
Required in VegBranch: Not required. Only necessary if you wish to assign plots to communities.
Represents VegBank Tables: commClass, classContributor (4), commInterpretation (4)
VegBranch can generate for you: no
Questions: What communities do each plot belong to?
According to whom?
With what degree of certainty?
Expertise Level: advanced

Fields (Columns) in this table:

» Required fields; you should always fill in values for these:
AuthorPlotCode
  text (100)		 Author's Plot number/code, or the original plot number if taken from literature. This "points" to the right plot in the Plot Data Loading Table.
classStartDate
  Date/Time		 Start date for this classification.
CommName1
  text (255)		 Name of a community which may be found in either your Community Concept Table or in the vegbank module. This field is limited to 255 characters, so if your community name is more than 255 characters, use the Community Concept Table.
InterpParty_A
  text (150)		 Primary Party: Link to a party that contributed to the classification event.
RoleCode_PartyA
  text (50)		 Role code for the contributor to this classification that is listed in InterpParty_A. Role Codes will be translated in the Translate Data Form.
» Best Practice fields:
classNotes
  Memo		 Notes about the classification event by the parties participating in the event. This might include the purpose or rationale for the classification.
ClassFit1
  Text (50)		 Indicates the degree of fit with the community concept being assigned in CommName1.
ClassConfidence1
  text (15)		 Indicates the degree of confidence of the interpretor(s) in the interpretation made for CommName1. This can reflect the level of familiarity with the classification or the sufficiency of information about the plot (e.g., High, Moderate, Low).
NotesInterp1
  Memo		 Notes pertaining to the classification analysis and decision, for CommName1.
