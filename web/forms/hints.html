Login:
The VegBank authentication system uses the user's email address as the 'Username'
for logging into the VegBank system.

The VegBank System, due to its developmental nature, started using a new
user authentication system in late Feb. 2002, and many of the user accounts
which did not contain the elements critical to this new system were removed
. If you find that you cannot login please create a new account. September 1, 2002 is
the date that marks the official kickoff date of the VegBank System and currently
any data that is resident in the system should be considered transient.

Nested Query:
General
The basic idea for using this query tool is to build a nested query by generating
A number of atomic qurey statements that are then nested by the application and
issued against the VegBank database. There is a bug in the current system that allows
for only a single plant name (the 'Plant Taxon' query criteria) to be issued against
the database for each nested query.

Download:
ascii format
This will download a result set as an ASCII data set where the site data related
to the plots will be in one entity and the species-related data will be in a second.

Nested Query:
Criteria
The query criteria element is the database attribute that is used for selecting data
from the VegBank system. Currently there is only a small number of query criteria elements
that can be used to construct a nested query, but in the future we anticipate extending this
number. Below is a small explanation of how to use each of the query criteria:

Plant Taxon: This criterion refers to a plant name associated with the plot to be targeted by the query. Currently, there can only be one plant taxon query element used in a single nested query. Usually this criterion is to be used with the 'like' operator and a value that is buffered by the ('%') VegBank wildcard. For example one might build a query element like:

Plant Taxon like Pinus%


State: This is the US state from which the plot was collected. The following query would return all the plot from the state of California

State is equal to CA


Elevation: This the topographic elevation of the plot in feet. The following query would return all the plots above 10000 feet.

Elevation is greater than 10000


Latitude: This is the latitude at which the plot was collected. The following query would return all the plots in the nothern hemisphere.

Latitude is greater than 0.0


Latitude: This is the longitude at which the plot was collected. The following query would return all the plots collected from the eastern hemisphere.

Longitude is greater than 0.0


Plot code: This is the code that the author of the plot used to describe the plot. The following query would return all the plots in colleceted during the TNC Yosemite plot collection project.

Plot code like %YOS%



Nested query:
Operator
The query operator is the expression used to limit the query by the query value.
The operators associated with numeric values are:

is greater than
is less than
is equal to

The operators used for string comparisons are:

like
contains


Nested query:
Value
This is the value to be applied to the query criteria by the operator. The following criteria require character values:

Plant Taxon
State
Plot Code
Geology
Plot Shape

The following query criteria require numeric values:

Latitude
Longitude
Elevation
