Login/Certification
Login:
The VegBank authentication system uses the user's email address as the 'Username' for logging into the VegBank system.

The VegBank System, due to its developmental nature, started using a new user authentication system in late Feb. 2002, and many of the user accounts which did not contain the elements critical to this new system were removed. If you find that you cannot login please create a new account. October 1, 2002 is the date that marks the official kickoff date of the VegBank System and currently any data that is resident in the system should be considered transient.

Importing data to VegBank

Loading data into VegBank:
General instructions
(Note: You must be a certified user to submit data to VegBank; see above for more on that.)

Currently, VegBank has been pre-loaded with a set of NPS (National Park Service) Plots, the USDA plant taxa concepts, and the NVC community concepts.

If you wish to load plot data of your own, you can do so by sending us your plot data in one of three formats, NPS MS-Access Database, VegBranch (VegBank's MS-Access Database), and VegBank native-XML.
(How? Login, then selects PLOTS, then LOAD Plot Data.)

If you wish to load plant or community data, you can specify the various data required through a series of webforms.
(How? Login, then selects PLANT TAXON / COMMUNITY TYPE, then Create/Enter New.)

Querying data in VegBank

Nested Query:
General
The basic idea for using this query tool is to build a nested query by generating a number of atomic (simple) query statements that are then combined and issued against the VegBank database.
There is currently a bug in the system that allows for only a single plant name (the 'Plant Taxon' query criteria) to be issued against the database for each nested query.

Dates:
Multiple options
Date often refers to the state of the database as of a certain date.
For example, one might query to see what taxa were in VegBank that matched "Quercus" as of 21-JUN-2001.

VegBank will accept dates in the following formats:

Aug 12, 2002
12-AUG-2002

Nested Query:
Criteria
The query criteria element is the database attribute that is used for selecting data from the VegBank system. Currently there is only a small number of query criteria elements that can be used to construct a nested query, but in the future we anticipate extending this number. Below is a small explanation of how to use each of the query criteria:
Plant Taxon: This criterion refers to a plant name associated with the plot to be targeted by the query. Currently, there can only be one plant taxon query element used in a single nested query. Usually this criterion is to be used with the 'like' operator and a value that is buffered by the ('%') VegBank wildcard. For example one might build a query element like:

Plant Taxon like Pinus%


State: This is the US state from which the plot was collected. The following query would return all the plot from the state of California

State is equal to CA


Elevation: This the topographic elevation of the plot in feet. The following query would return all the plots above 10000 feet.

Elevation is greater than 10000


Latitude: This is the latitude at which the plot was collected. The following query would return all the plots in the nothern hemisphere.

Latitude is greater than 0.0


Latitude: This is the longitude at which the plot was collected. The following query would return all the plots collected from the eastern hemisphere.

Longitude is greater than 0.0


Plot code: This is the code that the author of the plot used to describe the plot. The following query would return all the plots in colleceted during the TNC Yosemite plot collection project.

Plot code like %YOS%

Nested query:
Operator
The query operator is the expression used to limit the query by the query value.
The operators associated with numeric values are:

is greater than
is less than
is equal to

The operators used for string comparisons are:

like
contains

Value
